# so-arm100-ros2

ROS 2 workspace for the SO-ARM100 robotic arm. This repository currently includes the robot description, RViz visualization, `ros2_control` bringup, controller configuration, and a simple trajectory publisher for basic motion testing.

## Repository Layout

- `src/so_arm100_description`: URDF files, meshes, RViz config, and a description-only display launch file
- `src/so_arm_control`: controller YAML, bringup launch file, and a Python trajectory publisher node
- `build/`, `install/`, `log/`: generated by `colcon`

## Packages

### `so_arm100_description`

Owns the robot model and visualization assets:

- `urdf/so101_new_calib.urdf`: visualization model
- `urdf/so101_new_calib_control.urdf`: control-ready model
- `launch/display.launch.py`: starts `robot_state_publisher`, `joint_state_publisher_gui`, and RViz

### `so_arm_control`

Owns the control-side runtime pieces:

- `config/controllers.yaml`: `joint_state_broadcaster` and `arm_controller`
- `launch/bringup.launch.py`: starts `robot_state_publisher`, `ros2_control_node`, controller spawners, RViz, and the trajectory demo node
- `so_arm_control/arm_trajectory_publisher.py`: publishes alternating `JointTrajectory` targets to `/arm_controller/joint_trajectory`

## Build

From the workspace root:

```bash
colcon build --symlink-install
source install/setup.bash
```

## Run

Visualize the arm model in RViz:

```bash
ros2 launch so_arm100_description display.launch.py
```

Start the control bringup stack:

```bash
ros2 launch so_arm_control bringup.launch.py
```

Run the trajectory publisher by itself:

```bash
ros2 run so_arm_control arm_trajectory_publisher
```

## Test

Run the package tests and linters:

```bash
colcon test
colcon test-result --verbose
```

## Current Scope

This is a learning-oriented ROS 2 project focused on:

- ROS 2 workspace structure
- URDF-based robot description
- RViz visualization
- `ros2_control` bringup
- controller configuration
- simple trajectory publishing

The `my_robot_controller` tutorial package is intentionally not part of the main project.
